# 家庭网部署：让家人连家里 WiFi 就能用 App

像应用市场里的应用一样：**你在一台电脑上跑后端，把 APK 发给家人安装，大家连家里 WiFi 即可使用，无需每人再填服务器地址。**

---

## 一、思路概览

1. **家里有一台电脑长期（或需要用时）开机**，在这台电脑上运行本项目的后端。
2. **给这台电脑设固定 IP**（如 `192.168.1.100`），这样手机、平板都能稳定访问。
3. **后端监听整个局域网**（`0.0.0.0:8000`），允许同 WiFi 下的设备访问。
4. **打包 APK 时把“服务器地址”写死为这台电脑的地址**（如 `http://192.168.1.100:8000/api`），装到任何人的手机上都会自动连你家这台电脑，无需再在 App 里设置。

---

## 二、步骤 1：给运行后端的电脑设固定 IP

- **路由器里绑定 IP（推荐）**  
  登录路由器管理页（一般是 `192.168.1.1` 或 `192.168.0.1`），找到「DHCP 静态分配 / 静态 IP / 地址保留」等，把运行后端的电脑的 **MAC 地址** 绑定到一个固定 IP，例如 `192.168.1.100`。  
  这样每次这台电脑连上家里 WiFi，都会拿到同一个 IP。

- **或在电脑上手动设静态 IP**  
  在系统网络设置里，把该电脑的 IPv4 设为固定地址（如 `192.168.1.100`）、子网掩码和网关按你家路由器填写。  
  设好后记下这个 IP，后面打包和文档里都用它。

---

## 三、步骤 2：在这台电脑上启动后端

在这台电脑上打开项目，进入后端目录并启动：

```bash
cd backend
# Windows:
venv\Scripts\activate
python manage.py runserver 0.0.0.0:8000

# macOS/Linux:
source venv/bin/activate
python manage.py runserver 0.0.0.0:8000
```

- **`0.0.0.0:8000`** 表示监听本机所有网卡，同一 WiFi 下的手机、平板都能通过 `http://你的电脑IP:8000` 访问。
- 若关闭了 DEBUG，需要允许跨域。在**同一终端**里可先设置环境变量再启动：

```bash
# Windows PowerShell：
$env:DJANGO_DEBUG="True"
python manage.py runserver 0.0.0.0:8000

# 或者允许所有 CORS（仅限家庭内网）：
$env:CORS_ALLOW_ALL_ORIGINS="True"
python manage.py runserver 0.0.0.0:8000
```

```bash
# macOS/Linux：
export DJANGO_DEBUG=True
python manage.py runserver 0.0.0.0:8000
```

这样家人手机上的 App（来自你的 APK）请求 `http://你的电脑IP:8000` 时，后端会正常响应。

---

## 四、步骤 3：用“家里电脑的地址”打包 APK（关键）

在**你用来打包的那台电脑**上：

1. 进入前端目录，根据**运行后端的电脑的固定 IP** 建一个生产环境用的环境变量文件。

   - 复制示例：`frontend/.env.production.example` → `frontend/.env.production`
   - 打开 `frontend/.env.production`，把其中的 `192.168.1.100` 改成你家的固定 IP，例如：

   ```env
   VITE_API_BASE=http://192.168.1.100:8000/api
   ```

   这里的 IP 必须是**步骤 1 里给“运行后端的电脑”设的固定 IP**。

2. 再按项目里的 Android 打包流程打 APK，例如：

   ```bash
   cd frontend
   npm run build:app
   cd android
   .\gradlew.bat assembleDebug
   ```

   或使用项目里已有的 `npm run apk` 等脚本。

这样打出来的 APK 里，**默认请求的服务器地址**就是 `http://192.168.1.100:8000/api`，不再依赖用户在 App 里手动填。

---

## 五、步骤 4：把 APK 发给家人使用

- 将生成的 **`app-debug.apk`**（一般在 `frontend/android/app/build/outputs/apk/debug/`）用微信、网盘等方式发给家人。
- 家人安装后，**手机连上你家同一个 WiFi**，打开 App 即可使用，**无需再点“设置 → 服务器地址”**。
- 若某台手机暂时连的是流量或别的 WiFi，会连不上；连回家里 WiFi 即可。

---

## 六、以后 IP 变了怎么办

- **推荐**：在路由器里一直用“静态 IP 绑定”，尽量不换这台电脑的 IP，一次配置长期有效。
- 若换了路由器或不得不改 IP：
  - **做法 A**：用新 IP 重新建一次 `.env.production`，再打一版 APK 发给家人覆盖安装。
  - **做法 B**：让家人在 App 里点 **设置 → 服务器地址**，改成新地址（如 `http://192.168.1.101:8000/api`）保存，无需重装。

---

## 七、小结

| 项目       | 说明 |
|------------|------|
| 后端运行在哪 | 家里一台电脑，固定 IP（如 192.168.1.100） |
| 后端怎么启动 | `python manage.py runserver 0.0.0.0:8000`，必要时设 CORS 或 DEBUG |
| APK 里写什么 | 打包前在 `frontend/.env.production` 里设 `VITE_API_BASE=http://固定IP:8000/api` |
| 家人怎么用   | 安装你发的 APK，连家里 WiFi，打开即用，像应用市场里的应用一样 |

这样，别的手机只要连你家 WiFi，装你的 APK，就能像用普通应用一样使用，无需每人自己填服务器地址。
