# 公网部署：让全国各地的人都能用你的 App

目标是：**后端放在公网服务器上，任何人装上你的 APK、有网就能用**，不限于同一 WiFi。

---

## 一、整体思路

| 环节 | 说明 |
|------|------|
| **后端** | 部署到一台**有公网 IP 的服务器**（云服务器 / VPS），用域名或 IP 提供 API（建议 HTTPS）。 |
| **App** | 打包时把「服务器地址」写成你的**公网 API 地址**，装到任何人的手机上都会连你这台服务器。 |
| **分发** | 把 APK 发给别人（链接、网盘、应用商店等），对方安装后只要有网络就能用。 |

---

## 二、准备公网服务器

需要一台能从外网访问的机器，例如：

- **国内**：阿里云 ECS、腾讯云 CVM、华为云等（需备案域名若用 80/443）
- **国外**：AWS、DigitalOcean、Vultr 等（可用 IP 或域名，无需备案）

得到：

- **公网 IP**（如 `1.2.3.4`），或  
- **域名**（如 `api.yourname.com`，并解析到该 IP）

下面以「有域名 + HTTPS」为例；若暂时只有 IP，可先把域名相关步骤换成 `http://你的公网IP:8000`。

---

## 三、在服务器上部署后端（Django）

### 1. 安装环境

在服务器上安装 Python 3.10+、Nginx（可选，用于反向代理和 HTTPS）。把本项目代码上传到服务器（git clone 或打包上传）。

### 2. 创建生产环境变量

在服务器上、项目根目录或 `backend` 同级的 `.env` 或系统环境变量里配置，例如：

```bash
# 必须改：随机长字符串
export DJANGO_SECRET_KEY="你的随机密钥"

# 生产建议关闭 DEBUG
export DJANGO_DEBUG=False

# 你的域名或公网 IP，逗号分隔多个
export DJANGO_ALLOWED_HOSTS="api.yourname.com,1.2.3.4"

# 允许 App 请求（Capacitor/手机端来源）
export CORS_ALLOW_ALL_ORIGINS=True
# 或只允许你的域名（若用网页管理端）：
# export CORS_ALLOWED_ORIGINS="https://api.yourname.com,https://yourname.com"
```

数据库若用 SQLite（小规模可先用）：

- 保证 `backend` 目录可写，`db.sqlite3` 会生成在该目录。

若用 MySQL：

```bash
export DB_ENGINE=django.db.backends.mysql
export DB_NAME=数据库名
export DB_USER=用户
export DB_PASSWORD=密码
export DB_HOST=127.0.0.1
export DB_PORT=3306
```

### 3. 用 Gunicorn 跑 Django

在服务器上进入项目目录，例如：

```bash
cd /path/to/ElderlyCarePlatform/backend
python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
python manage.py migrate
python manage.py collectstatic --noinput
gunicorn config.wsgi:application -b 0.0.0.0:8000 -w 2
```

这样后端在 8000 端口监听。**生产环境建议前面加 Nginx 反向代理并配 HTTPS**（见下）。

### 4. Nginx + HTTPS（推荐）

用 Nginx 把 80/443 转到本机 8000，并配 SSL（如 Let's Encrypt）：

```nginx
server {
    listen 80;
    server_name api.yourname.com;
    return 301 https://$host$request_uri;
}
server {
    listen 443 ssl;
    server_name api.yourname.com;
    ssl_certificate     /path/to/fullchain.pem;
    ssl_certificate_key /path/to/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

证书可用 certbot（Let's Encrypt）免费申请。配置好后，API 基地址为：**`https://api.yourname.com`**（若 API 挂在根路径）或 **`https://api.yourname.com/api`**（若 Nginx 或 Django 做了 `/api` 前缀）。

## 四、确认 API 基地址

- 若 Django 的接口都在根路径（如 `/api/auth/login/`），且 Nginx 直接 `proxy_pass http://127.0.0.1:8000`，则 API 基地址一般为：  
  **`https://api.yourname.com/api`**（前提是 Django 的 `urls.py` 里把接口都挂在 `api/` 下）。
- 若你项目里接口前缀就是 `/api/`，那前端请求的 base 就应是 **`https://api.yourname.com/api`**，不要末尾再加一次 `/api`。

在服务器上可用 curl 自测：

```bash
curl -I https://api.yourname.com/api/health/
```

返回 200 或 401 等说明接口可达。

## 五、打包 App：写死公网 API 地址

在你**本地**打包 APK 时，让应用默认连你的公网后端：

1. 在 **frontend** 目录下创建或修改 **`.env.production`**：

   ```env
   VITE_API_BASE=https://api.yourname.com/api
   ```

   把 `api.yourname.com` 换成你的实际域名；若暂时用 IP：`http://你的公网IP:8000/api`（建议尽快换成 HTTPS）。

2. 打 APK：

   ```bash
   cd frontend
   npm run apk
   ```

生成的 **app-debug.apk** 里已经内置了上面的地址，**任何人安装后只要有网络（WiFi 或 4G/5G）都会连你这台服务器**，全国各地都能用。

---

## 六、分发 APK

- **方式一**：把 `app-debug.apk` 放到网盘（阿里云盘、百度网盘等），发链接或二维码给用户下载安装。  
- **方式二**：放到你自己网站提供下载。  
- **方式三**：上架应用商店（华为、小米、应用宝等），需按各商店要求做签名、隐私政策等，审核通过后用户可从商店安装。

只要 App 里写的是你的公网 API 地址，**不限于同一 WiFi，全国各地都能用**。

---

## 七、安全检查清单（生产环境）

- [ ] `DJANGO_DEBUG=False`
- [ ] `DJANGO_SECRET_KEY` 使用随机强密钥，不要用默认
- [ ] `ALLOWED_HOSTS` 只包含你的域名/IP，不要用 `*`
- [ ] 使用 HTTPS，不要在生产环境用 HTTP
- [ ] 数据库若用 MySQL，使用强密码、不对外网开放 3306
- [ ] 定期备份数据库和重要配置

---

## 八、小结

| 步骤 | 内容 |
|------|------|
| 1 | 准备一台有公网 IP 的服务器，可选域名 + HTTPS |
| 2 | 在服务器上部署 Django（Gunicorn + Nginx），配置好 ALLOWED_HOSTS、CORS、SECRET_KEY |
| 3 | 确认 API 可访问：`https://你的域名/api/...` |
| 4 | 本地在 `frontend/.env.production` 里设 `VITE_API_BASE=https://你的域名/api`，执行 `npm run apk` |
| 5 | 把 APK 通过链接或应用商店分发给用户 |

这样你的应用就和「应用市场里那些应用」一样：**全国各地的人只要装上你的 APK、有网络就能用**，后端在你自己的公网服务器上。
