# 手机端无法访问 https://47.111.26.171 的修复

电脑能访问、手机不行的原因：**自签名证书必须与 App 内嵌的证书完全一致**，否则手机 App 会拒绝连接。

## 一键修复（按顺序执行）

### 步骤 1：本地重新生成证书（证书 + App 内嵌同步）

```bash
cd frontend
node scripts/generate-self-signed-cert.js
```

这会同时更新：
- `deploy/certs/`（供服务器用）
- `frontend/android/.../res/raw/server_crt`（供 App 信任用）

### 步骤 2：上传 certs 到服务器

把 `deploy/certs/` 目录上传到服务器的 `~/ElderlyCarePlatform/deploy/certs/`，覆盖原有文件。

```bash
# 在本地执行（PowerShell，把 47.111.26.171 换成你的服务器 IP）：
scp -r deploy/certs/* root@47.111.26.171:~/ElderlyCarePlatform/deploy/certs/
```

### 步骤 3：服务器切换到 HTTPS 并重启前端

```bash
# SSH 登录服务器后执行：
cd ~/ElderlyCarePlatform
docker compose -f docker-compose.yml -f deploy/docker-compose.https-self-signed.yml up -d frontend
```

确认阿里云安全组已放行 **443** 端口。

### 步骤 4：本地重新打 APK

```bash
cd frontend
npm run apk
```

安装新的 `app-debug.apk` 到手机，卸载旧版本再装，确保用的是最新证书。

---

## 若仍不行：验证证书是否一致

**在服务器上**查看证书指纹：

```bash
openssl x509 -in ~/ElderlyCarePlatform/deploy/certs/fullchain.pem -noout -fingerprint -sha256
```

**在本地**查看 App 内嵌证书指纹：

```bash
openssl x509 -in frontend/android/app/src/main/res/raw/server_crt -noout -fingerprint -sha256
```

两个指纹必须**完全一致**。若不一致，重新执行步骤 1～4。
